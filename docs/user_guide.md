# BioFlowGBIF 사용자 가이드

버전: 2.0.0

BioFlowGBIF는 엑셀 또는 CSV 데이터를 GBIF 업로드용 Darwin Core 형식으로 매핑하고, 결과를 엑셀 파일로 저장하는 Windows 데스크톱 프로그램입니다.
v2.0.0부터는 GBIF occurrence 데이터를 직접 조회해 지도와 그래프로 분석하고, DOI 다운로드 요청과 QGIS용 GeoJSON 저장도 지원합니다.

## 사용 전 준비

- `BioFlowGBIF.exe`를 실행합니다.
- 엑셀 또는 CSV 파일을 준비합니다.
- 지도 미리보기를 사용하려면 인터넷 연결이 필요합니다.
- 현재 기본 기능과 좌표 지도 미리보기에는 R 또는 Rscript 설치가 필요하지 않습니다.

## 기본 작업 흐름

1. `파일 업로드` 버튼을 누릅니다.
2. 사용할 엑셀 또는 CSV 파일을 선택합니다.
3. 엑셀 파일인 경우 시트 선택 창에서 변환할 시트를 선택합니다.
4. 왼쪽 원본 테이블에서 헤더로 사용할 행을 클릭합니다.
5. `선택 행을 헤더로 적용` 버튼을 누릅니다.
6. 필요한 경우 `원본 좌표 확인`으로 매핑 전 좌표 위치를 먼저 확인합니다.
7. 오른쪽 `GBIF 필드 매핑` 영역에서 자동 매핑된 컬럼을 확인합니다.
8. 잘못 매핑된 항목이 있으면 직접 올바른 원본 컬럼을 선택합니다.
9. 필요한 GBIF 컬럼이 더 있으면 `컬럼 추가` 버튼으로 추가합니다.
10. `결과 확인` 버튼을 눌러 변환 결과를 생성합니다.
11. 결과 테이블에서 값을 확인하고 필요한 경우 직접 수정합니다.
12. `빈칸 점검` 영역에서 비어 있는 값의 위치를 확인합니다.
13. `좌표 확인` 버튼으로 매핑 결과의 좌표를 지도에서 확인합니다.
14. `매핑 결과 엑셀 저장` 버튼으로 결과 파일을 저장합니다.

## GBIF 종 데이터 분석

`GBIF 분석` 탭에서는 찾는 종의 GBIF occurrence 좌표 데이터를 가져와 지도와 그래프로 확인할 수 있습니다.

사용 방법:

1. 학명 또는 종명을 입력합니다.
2. 필요한 경우 국가코드(`KR`, `JP`, `US` 등)를 입력합니다.
3. 빠른 분석으로 가져올 데이터 건수를 선택합니다.
4. 그래프 기준을 `종합`, `연도별`, `월별`, `계절별` 중 선택합니다.
5. 그래프 모양을 `막대 그래프` 또는 `선 그래프`로 선택합니다.
6. 필요한 경우 연도 범위와 월 범위를 지정합니다.
7. `GBIF 가져오기`를 눌러 데이터를 조회합니다.
8. `지도/그래프 열기`로 HTML 리포트를 확인합니다.

`종합` 모드는 한 리포트 안에 연도별, 월별, 계절별 그래프를 함께 표시합니다. 월별/계절별 그래프는 연도 범위 안의 각 연도를 여러 선 또는 막대로 비교할 수 있습니다.

빠른 분석은 GBIF occurrence search API를 사용하며, 앱에서는 최대 100,000건까지 조회할 수 있습니다. 논문용 전체 데이터와 DOI가 필요한 경우 `DOI 다운로드 요청`을 사용하세요.

## GBIF DOI 다운로드 요청

논문이나 보고서에 GBIF 데이터를 인용하려면 GBIF occurrence download DOI가 필요합니다.

사용 방법:

1. GBIF 분석 조건을 입력합니다.
2. 연도 범위와 월 범위를 필요한 만큼 지정합니다.
3. GBIF username, email, password를 입력합니다.
4. `DOI 다운로드 요청`을 누릅니다.
5. 요청이 성공하면 다운로드 URL이 화면에 표시되고, GBIF가 파일 준비 완료 알림을 이메일로 보냅니다.
6. 다운로드 페이지에서 공식 DOI와 citation 문구를 확인합니다.

비밀번호는 앱에 저장하지 않습니다. username과 email만 다음 입력 편의를 위해 저장됩니다.

## GBIF 분석 결과 저장

GBIF 분석 데이터는 아래 형식으로 저장할 수 있습니다.

- `CSV 저장`: occurrence 테이블을 CSV로 저장합니다.
- `엑셀 저장`: occurrence 데이터, 요약 정보, 그래프용 집계표, Excel 차트를 함께 저장합니다.
- `QGIS 저장`: QGIS에서 바로 열 수 있는 GeoJSON 파일로 좌표 데이터를 저장합니다.

`QGIS 저장`은 유효한 위도/경도 좌표만 GeoJSON Point로 저장합니다. QGIS에서 GeoJSON을 열면 배경지도, 심볼, 레이아웃을 설정한 뒤 PNG/PDF/SVG 지도로 출력할 수 있습니다.

## 파일 업로드와 헤더 선택

`파일 업로드`를 누르면 엑셀 또는 CSV 파일을 선택할 수 있습니다. 한 번 파일을 불러온 뒤에는 다음 파일 선택 창이 마지막으로 사용한 폴더에서 열립니다.

파일을 선택하면 원본 데이터가 왼쪽 테이블에 표시됩니다. 실제 컬럼명이 들어 있는 행을 클릭한 뒤 `선택 행을 헤더로 적용`을 누르면 해당 행이 컬럼명으로 사용됩니다.

헤더를 적용해야 매핑, 원본 좌표 확인, 결과 확인 기능을 사용할 수 있습니다.

## 원본 좌표 확인

`원본 좌표 확인`은 GBIF 매핑을 하기 전에 원본 엑셀/CSV 파일의 위도와 경도만 먼저 지도에서 확인하는 기능입니다.

사용 방법:

1. 파일을 업로드합니다.
2. 헤더 행을 선택하고 `선택 행을 헤더로 적용`을 누릅니다.
3. `원본 좌표 확인` 버튼을 누릅니다.
4. 위도 컬럼을 선택합니다.
5. 경도 컬럼을 선택합니다.
6. 생성된 지도 미리보기에서 좌표 위치를 확인합니다.

컬럼명이 `위도`, `latitude`, `lat`, `경도`, `longitude`, `lon`, `lng` 등으로 되어 있으면 프로그램이 기본 선택값을 자동으로 맞추려고 시도합니다. 실제 데이터에 맞지 않으면 직접 다른 컬럼을 선택하면 됩니다.

위도와 경도는 서로 다른 컬럼을 선택해야 합니다. 좌표가 비어 있거나 범위를 벗어난 값은 지도에 표시되지 않고 제외 개수로 집계됩니다.

## GBIF 필드 매핑

헤더를 적용하면 오른쪽 `GBIF 필드 매핑` 영역에 기본 Darwin Core 필드가 표시됩니다.

기본 제공 필드:

- `occurrenceID`
- `basisOfRecord`
- `eventDate`
- `institutionCode`
- `scientificName`
- `collectionCode`
- `decimalLatitude`
- `decimalLongitude`
- `countryCode`
- `dateIdentified`
- `identifiedBy`
- `stateProvince`
- `county`
- `locality`
- `verbatimLocality`
- `catalogNumber`
- `vernacularName`

프로그램은 원본 컬럼명과 자동 매핑 후보를 비교해 가능한 항목을 미리 선택합니다. 자동 선택이 틀렸다면 드롭다운에서 올바른 원본 컬럼을 선택합니다.

`basisOfRecord`는 반드시 선택해야 합니다. `countryCode`, `collectionCode`처럼 직접 입력값을 사용할 수 있는 항목은 드롭다운 선택 대신 입력칸에 값을 넣을 수 있습니다.

## 컬럼 추가

기본 필드 외에 더 필요한 Darwin Core 필드는 `컬럼 추가` 버튼으로 추가할 수 있습니다.

추가 가능한 예:

- `recordedBy`
- `individualCount`
- `habitat`
- `samplingProtocol`
- `coordinateUncertaintyInMeters`
- `kingdom`
- `phylum`
- `class`
- `order`
- `family`
- `genus`
- `taxonRank`
- `scientificNameAuthorship`

추가한 필드도 기존 필드와 동일하게 원본 컬럼을 선택해 매핑합니다.

## 결과 확인과 수정

`결과 확인`을 누르면 매핑 결과가 아래 결과 테이블에 표시됩니다.

결과 테이블에서는 값을 직접 수정할 수 있습니다. 저장 전에 날짜, 좌표, 필수값, 오탈자 등을 확인하세요.

빈칸이 있는 경우 `빈칸 점검` 영역에 컬럼별 빈칸 개수와 일부 위치가 표시됩니다. 위치 버튼을 누르면 해당 셀로 이동해 값을 확인할 수 있습니다.

## 매핑 결과 좌표 확인

`좌표 확인`은 `결과 확인` 이후 사용할 수 있습니다. 매핑 결과의 `decimalLatitude`, `decimalLongitude` 값을 기준으로 지도 미리보기를 생성합니다.

`원본 좌표 확인`과의 차이:

- `원본 좌표 확인`: 매핑 전 원본 파일의 위도/경도 컬럼을 직접 골라 확인합니다.
- `좌표 확인`: 매핑 후 결과 테이블의 `decimalLatitude`, `decimalLongitude`를 확인합니다.

지도 미리보기 파일은 아래 위치에 생성됩니다.

```text
data\output\coordinate_preview_map.html
```

지도에서는 유효한 좌표만 표시됩니다. 마커를 클릭하면 행 번호, 위도, 경도, 학명, 일자, 지역 등 주요 정보를 확인할 수 있습니다.

좌표가 비어 있거나 숫자로 변환할 수 없거나 허용 범위를 벗어나면 제외됩니다. 제외된 좌표가 있으면 알림창과 지도 우측 요약에 행 번호가 표시되고, 전체 목록은 `data\output\coordinate_invalid_rows.csv`에 저장됩니다.

## 결과 저장

결과를 확인하고 필요한 수정을 마쳤다면 `매핑 결과 엑셀 저장` 버튼을 누릅니다.

저장 위치와 파일명을 선택하면 현재 결과 테이블의 내용이 엑셀 파일로 저장됩니다.
프로그램은 마지막으로 저장한 폴더를 기억하고, 다음 저장 때 같은 폴더를 기본 위치로 엽니다.

## 자주 확인할 사항

- 헤더 행을 올바르게 선택했는지 확인합니다.
- `basisOfRecord`가 선택되어 있는지 확인합니다.
- `decimalLatitude`, `decimalLongitude`가 올바른 원본 컬럼에 연결되어 있는지 확인합니다.
- 좌표값이 위도 `-90`에서 `90`, 경도 `-180`에서 `180` 범위 안에 있는지 확인합니다.
- 저장 전 빈칸 점검 결과를 확인합니다.
- 지도 미리보기가 보이지 않으면 인터넷 연결을 확인합니다.
